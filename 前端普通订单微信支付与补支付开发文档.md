# 前端普通订单微信支付与补支付开发文档

本文档用于指挥微信小程序前端 agent 开发普通点单微信支付与补支付能力。以当前后端实现为准，适用于用户在菜单/购物车提交普通订单、调起微信支付、订单详情继续支付或重新支付的完整链路。

## 1. 目标

- 普通点单创建订单后，使用后端返回的 `payParams` 调起 `wx.requestPayment`。
- 支付完成后，以后端订单详情的 `paymentStatus=PAID` 作为最终成功依据。
- 用户退出支付页、取消支付或支付失败后，可在订单详情页点击“继续支付”或“重新支付”再次拉起支付。
- 补支付只复用原订单，不重新下单，不重新扣库存，不重新占用会员卡。

## 2. 核心状态

订单有两个状态字段，前端必须分开展示和判断：

| 字段 | 含义 |
| --- | --- |
| `status` | 制作流程状态：`PENDING`、`ACCEPTED`、`MAKING`、`READY`、`COMPLETED`、`CANCELLED` |
| `paymentStatus` | 支付状态：`UNPAID`、`PENDING`、`PAID`、`FAILED`、`CLOSED` |

支付按钮显示规则：

| 条件 | 展示 |
| --- | --- |
| `paymentStatus=PENDING` 且 `status!=CANCELLED` | 继续支付 |
| `paymentStatus=FAILED` 且 `status!=CANCELLED` | 重新支付 |
| `paymentStatus=PAID` | 不显示支付按钮 |
| `paymentStatus=CLOSED` 或 `status=CANCELLED` | 不显示支付按钮 |

建议文案：

| `paymentStatus` | 文案 |
| --- | --- |
| `PENDING` | 支付确认中 |
| `PAID` | 已支付 |
| `FAILED` | 支付失败 |
| `CLOSED` | 支付已关闭 |
| `UNPAID` | 未支付 |

## 3. 接口

所有订单接口都需要用户 token：

```http
Authorization: Bearer <token>
```

### 3.1 创建订单

```http
POST /api/v1/orders
Content-Type: application/json
Authorization: Bearer <token>
```

请求：

```json
{
  "items": [
    {
      "productId": "00000000-0000-0000-0000-000000000001",
      "quantity": 2
    }
  ],
  "remark": "少冰",
  "memberCardId": "30000000-0000-0000-0000-000000000001"
}
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `items` | array | 是 | 至少 1 项 |
| `items[].productId` | UUID | 是 | 商品 ID |
| `items[].quantity` | number | 是 | 最小 1 |
| `remark` | string/null | 否 | 最长 255 |
| `memberCardId` | UUID/null | 否 | 使用会员卡优惠时传入；不用会员卡时不传或传 `null` |

响应 `data`: `OrderResponse`

```json
{
  "id": "00000000-0000-0000-0000-000000000001",
  "pickupNo": 12,
  "originalTotalCents": 5600,
  "discountCents": 0,
  "totalCents": 5600,
  "status": "PENDING",
  "paymentStatus": "PENDING",
  "outTradeNo": "ODxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
  "remark": "少冰",
  "createdAt": "2026-06-10T16:30:00+08:00",
  "paidAt": null,
  "payParams": {
    "appId": "wx123456",
    "timeStamp": "1780900000",
    "nonceStr": "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
    "packageValue": "prepay_id=wx201410272009395522657a690389285100",
    "signType": "RSA",
    "paySign": "base64-signature"
  },
  "memberCard": null,
  "items": [
    {
      "productId": "00000000-0000-0000-0000-000000000001",
      "productName": "拿铁",
      "unitPriceCents": 2800,
      "quantity": 2,
      "subtotalCents": 5600
    }
  ]
}
```

说明：

- 创建成功且需要支付时，后端返回 `paymentStatus=PENDING` 和 `payParams`。
- 如果订单实付金额为 0，可能返回 `paymentStatus=PAID` 且 `payParams=null`，前端不要调起支付。
- 订单列表和订单详情通常不会重新返回可用的 `payParams`；继续支付必须调用补支付接口。

### 3.2 继续支付/重新支付

```http
POST /api/v1/orders/{id}/pay
Authorization: Bearer <token>
```

响应 `data`: `OrderResponse`

用途：

- 用户创建订单后退出支付页。
- 用户取消 `wx.requestPayment`。
- 用户支付失败后希望再次支付。

后端规则：

- 只允许当前登录用户操作自己的订单。
- 只允许 `paymentStatus=PENDING` 或 `FAILED` 的订单继续支付。
- `FAILED` 订单重新拉起支付后会恢复为 `PENDING`。
- `paymentStatus=PAID` 会返回业务错误“订单已支付”。
- `paymentStatus=CLOSED` 或 `status=CANCELLED` 会返回业务错误“订单已关闭，不能继续支付”。
- 该接口沿用原订单和原 `outTradeNo`，不会重新扣库存或重新使用会员卡。

### 3.3 获取订单详情

```http
GET /api/v1/orders/{id}
Authorization: Bearer <token>
```

响应 `data`: `OrderResponse`

用途：

- 支付后刷新最终支付状态。
- 订单详情页展示订单状态、支付状态、取餐号、商品快照、会员卡优惠和金额。

### 3.4 获取订单列表

```http
GET /api/v1/orders
Authorization: Bearer <token>
```

响应 `data`: `OrderResponse[]`

用途：

- 我的订单页展示历史订单。
- 列表项要展示 `paymentStatus`，并按按钮规则显示“继续支付/重新支付”入口。

## 4. wx.requestPayment 映射

后端字段和微信小程序字段映射如下：

| 后端字段 | wx.requestPayment 字段 |
| --- | --- |
| `payParams.timeStamp` | `timeStamp` |
| `payParams.nonceStr` | `nonceStr` |
| `payParams.packageValue` | `package` |
| `payParams.signType` | `signType` |
| `payParams.paySign` | `paySign` |

示例封装：

```js
function requestWechatPayment(payParams) {
  return new Promise((resolve, reject) => {
    if (!payParams) {
      resolve({ skipped: true })
      return
    }
    wx.requestPayment({
      timeStamp: payParams.timeStamp,
      nonceStr: payParams.nonceStr,
      package: payParams.packageValue,
      signType: payParams.signType,
      paySign: payParams.paySign,
      success: resolve,
      fail: reject
    })
  })
}
```

注意：

- 不要把 `packageValue` 改名传给后端；只是在调用 `wx.requestPayment` 时映射为 `package`。
- `wx.requestPayment` 成功回调不等于后端已发出支付成功通知，必须刷新订单详情确认。
- `wx.requestPayment` 可能触发页面 `onHide`，不要在 `onHide` 停止订单状态刷新逻辑；需要离开页面时再在 `onUnload` 清理定时器。

## 5. 前端流程

### 5.1 创建订单并支付

1. 用户从购物车/菜单页提交订单，调用 `POST /api/v1/orders`。
2. 后端返回 `OrderResponse`。
3. 若 `data.paymentStatus === 'PAID'` 或 `data.payParams` 为空，直接进入订单成功/详情页。
4. 若存在 `data.payParams`，立即调用 `wx.requestPayment`。
5. 支付回调后刷新订单详情：
   - 建议立即调用一次 `GET /api/v1/orders/{id}`。
   - 若仍为 `PENDING`，每 2 秒轮询一次，最多 5 次。
   - 仍未变为 `PAID` 时展示“支付结果确认中，请稍后刷新”。
6. 支付失败或用户取消时，保留订单并跳转/停留到订单详情页，展示“继续支付”。

### 5.2 继续支付/重新支付

1. 订单详情页或订单列表页判断按钮：
   - `PENDING` 展示“继续支付”。
   - `FAILED` 展示“重新支付”。
2. 用户点击按钮，调用 `POST /api/v1/orders/{id}/pay`。
3. 后端返回新的 `payParams`。
4. 调用 `wx.requestPayment`。
5. 支付后刷新订单详情，仍以 `paymentStatus=PAID` 为最终成功依据。

### 5.3 支付失败处理

建议区分两类失败：

- 用户主动取消：提示“已取消支付，可在订单详情继续支付”。
- 其他失败：提示“支付未完成，请稍后重试”，并保留继续支付按钮。

不要在前端删除订单、重新下单或自动清空订单详情；后端已经创建订单并扣减库存。

## 6. 建议代码改动范围

如果当前小程序是原生微信小程序，建议按以下模块拆分：

- 订单服务层：新增 `retryOrderPayment(orderId)`，封装 `POST /api/v1/orders/{id}/pay`。
- 支付工具函数：封装 `requestWechatPayment(payParams)` 和支付后刷新/轮询逻辑。
- 下单入口：创建订单后调用统一支付函数，不再只跳转成功页。
- 订单详情页：展示 `paymentStatus`，增加继续支付/重新支付按钮。
- 订单列表页：展示支付状态，并对未支付订单提供继续支付入口。
- 全局/页面状态：支付完成后刷新订单详情；成功支付后如使用会员卡，触发会员卡缓存刷新。

如果项目已有以下文件，优先在现有结构内扩展，不要另起一套并行实现：

- `services/order.js`
- `pages/menu/menu.js`
- `pages/orders/detail.js`
- `pages/orders/index.js`
- `utils/request.js`

## 7. 错误处理

| 场景 | 前端处理 |
| --- | --- |
| 401 | 清理 token，重新微信登录，再让用户重试 |
| 403 | 提示“无权查看该订单” |
| 404 | 提示“订单不存在”并返回订单列表 |
| `订单已支付` | 刷新订单详情，隐藏支付按钮 |
| `订单已关闭，不能继续支付` | 刷新订单详情，隐藏支付按钮 |
| `微信支付下单失败` / `微信支付参数生成失败` | 保留订单详情页，提示稍后重试 |
| `wx.requestPayment:fail cancel` | 提示已取消支付，保留继续支付按钮 |

## 8. 验收标准

- 创建普通订单后能真实调起微信支付。
- 支付成功后，订单详情最终显示 `paymentStatus=PAID`。
- 用户取消支付后，订单详情/订单列表显示“继续支付”。
- 点击“继续支付”会调用 `POST /api/v1/orders/{id}/pay`，并再次调起微信支付。
- 支付失败订单显示“重新支付”，点击后可再次拉起支付，且订单不重复创建。
- `PAID/CLOSED/CANCELLED` 订单不显示支付按钮。
- 订单金额展示支持：
  - `originalTotalCents`
  - `discountCents`
  - `totalCents`
  - `memberCard.name`
  - `memberCard.discountProductName`
- 支付流程不因页面 `onHide` 中断；离开页面时在 `onUnload` 清理轮询。

## 9. 不在本次前端范围

- 不开发退款。
- 不开发用户取消订单接口。

- 不重新设计后端订单状态。
- 不用前端模拟支付成功替代 `wx.requestPayment`。
