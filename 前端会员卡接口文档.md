# 前端会员卡接口文档

接口基础地址沿用现有后端，所有用户态接口除微信支付回调外均需要：

```http
Authorization: Bearer <token>
```

响应包裹结构沿用：

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

## 1. 枚举

### 1.1 MemberCardType

| 值 | 含义 |
| --- | --- |
| `HALF_PRICE` | 每日半价卡 |
| `FREE_ITEM` | 每日免单卡 |

### 1.2 MemberCardSource

| 值 | 含义 |
| --- | --- |
| `PURCHASE` | 用户购买后自动发放 |
| `MANUAL` | 后台手动发放 |

### 1.3 MemberCardStatus

| 值 | 含义 |
| --- | --- |
| `ACTIVE` | 可用 |
| `DISABLED` | 已禁用 |

### 1.4 MemberCardPurchaseStatus

| 值 | 含义 |
| --- | --- |
| `PENDING` | 待支付或支付结果确认中 |
| `PAID` | 已支付并已发卡/续期 |
| `FAILED` | 支付失败 |
| `CLOSED` | 已关闭 |

## 2. 获取可购买会员卡模板

```http
GET /api/v1/member-card-templates
Authorization: Bearer <token>
```

响应 `data`: `MemberCardTemplateResponse[]`

```json
[
  {
    "id": "10000000-0000-0000-0000-000000000001",
    "name": "每日半价卡",
    "type": "HALF_PRICE",
    "priceCents": 990,

      {
        "id": "20000000-0000-0000-0000-000000000001",
        "name": "拿铁",
        "priceCents": 2800

    ]
  }
]
```

## 4. 创建会员卡购买单

```http
POST /api/v1/member-card-orders
Authorization: Bearer <token>
Content-Type: application/json
```

请求：

```json
{
  "templateId": "10000000-0000-0000-0000-000000000001"
}
```

响应 `data`: `MemberCardPurchaseResponse`

```json
{
  "id": "40000000-0000-0000-0000-000000000001",
  "templateId": "10000000-0000-0000-0000-000000000001",
  "templateName": "每日半价卡",
  "amountCents": 990,
  "status": "PENDING",
  "outTradeNo": "MCxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
  "payParams": {
    "appId": "wx123456",
    "timeStamp": "1780900000",
    "nonceStr": "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
    "packageValue": "prepay_id=wx201410272009395522657a690389285100",
    "signType": "RSA",
    "paySign": "base64-signature"
  }
}
```

小程序支付字段映射：

| 后端字段 | wx.requestPayment 字段 |
| --- | --- |
| `timeStamp` | `timeStamp` |
| `nonceStr` | `nonceStr` |
| `packageValue` | `package` |
| `signType` | `signType` |
| `paySign` | `paySign` |

## 5. 查询会员卡购买单状态

```http
GET /api/v1/member-card-orders/{id}
Authorization: Bearer <token>
```

响应 `data`: `MemberCardPurchaseStatusResponse`

```json
{
  "id": "40000000-0000-0000-0000-000000000001",
  "status": "PAID",
  "paidAt": "2026-06-08T10:05:00+08:00",
  "issuedCardId": "30000000-0000-0000-0000-000000000001"
}
```

## 6. 微信支付回调

该接口由微信支付调用，前端不直接调用。

```http
POST /api/v1/wechat-pay/notify/member-card-orders
Content-Type: application/json
```

后端处理成功时返回：

```json
{
  "code": "SUCCESS",
  "message": "成功"
}
```

## 7. 创建订单时使用会员卡

```http
POST /api/v1/orders
Authorization: Bearer <token>
Content-Type: application/json
```

请求新增可选字段 `memberCardId`：

```json
{
  "items": [
    {
      "productId": "20000000-0000-0000-0000-000000000001",
      "quantity": 2
    }
  ],
  "remark": "少冰",
  "memberCardId": "30000000-0000-0000-0000-000000000001"
}
```

响应 `data`: `OrderResponse`

```json
{
  "id": "50000000-0000-0000-0000-000000000001",
  "pickupNo": 12,
  "originalTotalCents": 5600,
  "discountCents": 1400,
  "totalCents": 4200,
  "status": "PENDING",
  "remark": "少冰",
  "createdAt": "2026-06-08T10:10:00+08:00",
  "memberCard": {
    "id": "30000000-0000-0000-0000-000000000001",
    "name": "每日半价卡",
    "discountProductName": "拿铁"
  },
  "items": [
    {
      "productId": "20000000-0000-0000-0000-000000000001",
      "productName": "拿铁",
      "unitPriceCents": 2800,
      "quantity": 2,
      "subtotalCents": 5600
    }
  ]
}
```

未使用会员卡时：

- `originalTotalCents` 等于 `totalCents`。
- `discountCents` 为 `0`。
- `memberCard` 为 `null`。

## 8. 错误场景

| HTTP | message | 场景 |
| --- | --- | --- |
| 400 | `会员卡已下架` | 购买已下架模板 |
| 400 | `微信支付下单失败` | 微信预支付创建失败 |
| 400 | `会员卡已失效` | 下单使用过期或禁用卡 |
| 400 | `该会员卡今日已使用` | 同一张卡当天重复使用 |
| 400 | `订单中没有该会员卡可用商品` | 购物车没有该卡适用商品 |
| 403 | `无权使用该会员卡` | 使用他人的会员卡 |
| 404 | `会员卡不存在` | 模板或用户卡 ID 不存在 |
| 404 | `会员卡购买单不存在` | 查询不存在的购买单 |

## 9. 金额规则

- 所有金额单位都是分。
- 一单最多使用一张会员卡。
- 一张卡只优惠一件商品。
- 多个商品符合条件时，后端自动选单价最高的一件。
- `HALF_PRICE`：优惠金额为单价除以 2，向下取整。
- `FREE_ITEM`：优惠金额为单价。
- 最终金额以 `totalCents` 为准。
