# 前端会员卡开发文档

本文面向 CttCafe 小程序前端 Agent，说明会员卡功能的页面、状态、流程和异常处理。后端以 JWT 鉴权，所有会员卡用户接口都需要携带 `Authorization: Bearer <token>`。

## 1. 页面结构

### 1.1 会员卡中心

建议新增“会员卡”入口，页面包含两个区域：

- 可购买会员卡：调用 `GET /api/v1/member-card-templates`，展示名称、类型、售价、有效天数、适用商品。
- 我的会员卡：调用 `GET /api/v1/member-cards`，展示卡名称、到期时间、今日是否已用、是否过期、适用商品。

会员卡类型展示建议：

- `HALF_PRICE`：每日半价卡。
- `FREE_ITEM`：每日免单卡。

### 1.2 结算页会员卡入口

确认订单或购物车结算区域增加“会员卡优惠”入口：

- 默认展示“未使用会员卡”。
- 根据当前购物车商品筛选可用卡。
- 只允许选择一张会员卡。
- 选中后展示预计优惠商品、预计优惠金额、预计实付金额。

预计优惠仅用于展示，最终金额以后端 `POST /api/v1/orders` 返回为准。

### 1.3 订单列表和订单详情

订单金额展示需要支持：

- `originalTotalCents`：原价。
- `discountCents`：优惠金额。
- `totalCents`：最终实付。
- `memberCard`：使用的会员卡摘要。

若 `discountCents > 0`，订单详情应展示原价、会员卡名称、优惠商品、优惠金额和实付金额。

## 2. 状态管理

建议维护以下状态：

- `memberCardTemplates`：可购买会员卡模板列表。
- `myMemberCards`：当前用户会员卡列表。
- `selectedMemberCardId`：结算页选中的会员卡 ID。
- `memberCardPurchaseOrder`：当前购买单状态和支付参数。
- `memberCardLoading`：模板、我的卡、购买单轮询等加载状态。

刷新时机：

- 登录成功后可预取我的会员卡。
- 进入会员卡中心时刷新模板和我的卡。
- 支付成功或购买单变为 `PAID` 后刷新我的会员卡。
- 下单成功后刷新我的会员卡，更新今日已用状态。
- 订单取消后刷新我的会员卡，后端会释放当日使用次数。

## 3. 购买流程

1. 用户在会员卡中心点击购买。
2. 前端调用 `POST /api/v1/member-card-orders`，传入 `templateId`。
3. 后端返回购买单和 `payParams`。
4. 小程序调用 `wx.requestPayment`：

```js
wx.requestPayment({
  timeStamp: payParams.timeStamp,
  nonceStr: payParams.nonceStr,
  package: payParams.packageValue,
  signType: payParams.signType,
  paySign: payParams.paySign
})
```

5. 支付完成后不要直接认为发卡成功，调用 `GET /api/v1/member-card-orders/{id}` 轮询状态。
6. 当购买单状态为 `PAID` 时，刷新 `GET /api/v1/member-cards`。

轮询建议：

- 支付成功回到页面后立即查一次。
- 若仍为 `PENDING`，每 2 秒查一次，最多 5 次。
- 超时仍未变为 `PAID` 时提示“支付结果确认中，请稍后刷新”。

## 4. 结算流程

### 4.1 可用卡筛选

前端可以用以下条件做初筛：

- `status === 'ACTIVE'`。
- `expired === false`。
- `usedToday === false`。
- 当前购物车至少包含一个会员卡 `products[].id` 中的商品。

后端仍会再次校验；前端筛选只用于减少无效选择。

### 4.2 预计优惠计算

一张会员卡只优惠一件适用商品：

- 在购物车中找出会员卡适用商品。
- 若有多个适用商品，选择单价最高的一件。
- `HALF_PRICE` 优惠金额为该商品单价的一半，向下取整。
- `FREE_ITEM` 优惠金额为该商品单价。

提交订单时把选中的会员卡 ID 放入 `memberCardId`：

```json
{
  "items": [
    {
      "productId": "00000000-0000-0000-0000-000000000001",
      "quantity": 2
    }
  ],
  "remark": "少冰",
  "memberCardId": "00000000-0000-0000-0000-000000000010"
}
```

### 4.3 下单后处理

以后端返回订单为准更新页面：

- 显示 `pickupNo`。
- 显示实付 `totalCents`。
- 若 `memberCard` 非空，展示会员卡优惠信息。
- 清空购物车并刷新我的会员卡。

## 5. 异常处理

常见错误消息和前端处理建议：

- `会员卡已失效`：提示用户重新选择会员卡并刷新我的会员卡。
- `该会员卡今日已使用`：取消选中该卡，刷新我的会员卡。
- `订单中没有该会员卡可用商品`：提示该卡不适用于当前购物车。
- `会员卡已下架`：购买入口刷新模板列表。
- `微信支付下单失败`：保留当前页面，允许用户重试购买。
- 购买单长时间 `PENDING`：展示“支付结果确认中”，提供手动刷新按钮。

## 6. 验收清单

- 可展示可购买会员卡。
- 可展示我的会员卡和今日使用状态。
- 可调起会员卡微信支付。
- 支付后可轮询购买单并刷新我的会员卡。
- 结算页可选择一张可用会员卡。
- 下单后订单金额展示原价、优惠、实付。
- 会员卡今日已使用后不能再次选择或下单使用。
